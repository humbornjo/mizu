# mizuoai

`mizuoai` combines Mizu route registration with OpenAPI 3.2 document
generation. It supports two handler styles through one document pipeline:

- typed handlers use reflected Go request and response types;
- raw `http.HandlerFunc` handlers attach an explicit OpenAPI operation without
  changing their transport behavior.

OpenAPI 3.2.0 is the default output version. OpenAPI 3.1 documents generated by
tools such as `protoc-gen-connect-openapi` can be preloaded or used as operation
sources.

## Installation

```bash
go get github.com/humbornjo/mizu/mizuoai
```

## Typed handlers

Typed handlers receive `Rx[I]` and `Tx[O]`. Request-envelope fields use their
JSON name to select a transport location. Inner fields use ordinary JSON names
for both runtime decoding and OpenAPI parameters.

```go
type GetUserRequest struct {
	Path struct {
		UserId string `json:"user_id" desc:"User identifier"`
	} `json:"path"`
	Query struct {
		Verbose bool `json:"verbose,omitempty"`
	} `json:"query"`
}

type User struct {
	Id   string `json:"id"`
	Name string `json:"name" minLength:"1"`
}

func handleGetUser(tx mizuoai.Tx[User], rx mizuoai.Rx[GetUserRequest]) {
	input, err := rx.MizuRead()
	if err != nil {
		http.Error(tx, err.Error(), http.StatusBadRequest)
		return
	}
	user := User{Id: input.Path.UserId, Name: "Mizu"}
	_ = tx.MizuWrite(&user)
}

srv := mizu.NewServer("users")
if err := mizuoai.Initialize(srv, "Users API",
	mizuoai.WithOaiInfoVersion("v1"),
	mizuoai.WithOaiDocumentation(),
); err != nil {
	panic(err)
}

mizuoai.Get(srv, "/users/{user_id}", handleGetUser,
	mizuoai.WithOperationOperationId("getUser"),
	mizuoai.WithOperationTags("users"),
	mizuoai.WithOperationSummary("Get a user"),
)
```

The supported request-envelope locations are:

- `json:"path"`
- `json:"query"`
- `json:"header"`
- `json:"body"`
- `json:"form"` for `multipart/form-data` or
  `application/x-www-form-urlencoded`

Use `contentType:"application/x-www-form-urlencoded"` on the outer form field
to document URL-encoded form data. Multipart is the default form media type.
The runtime decoder selects the matching decoder from the request's actual
`Content-Type`.

Legacy outer `mizu` tags and inner location-specific tags remain accepted, but
JSON names are the canonical form. Defining both forms with different values is
an initialization error.

## Reflected schemas and components

Named Go types are emitted once under `components.schemas` and referenced with
`$ref`. Reflection supports:

- booleans, strings, signed and unsigned integers, and floating-point values;
- aliases, pointers, arrays, slices, and maps with string keys;
- nested, shared, recursive, and mutually recursive structs;
- `time.Time`, `json.RawMessage`, and `[]byte`;
- JSON field names, `omitempty`, ignored fields, and explicit `required` tags;
- common schema tags such as `desc`, `deprecated`, `format`, `pattern`,
  `default`, `example`, `enum`, `minimum`, `maximum`, `minLength`, `maxLength`,
  `minItems`, and `maxItems`.

Parameter fields also accept `style`, `explode`, `allowReserved`,
`allowEmptyValue`, `example`, `required`, `deprecated`, and `desc`. Repeated
query or header values decode into slices and arrays.

Reflection supplies a sound default request and success-response schema. It
cannot infer error responses, security policy, content negotiation, examples,
callbacks, or protocol semantics. Use operation options or an authoritative
external operation for those fields.

Use `WithOaiSchemaName` when a named Go type needs a stable public component
name or two packages define the same Go type name:

```go
mizuoai.Initialize(srv, "Service API",
	mizuoai.WithOaiSchemaName[CreateUserRequest]("CreateUserRequest"),
)
```

## Complete operation options

Native `libopenapi` models can provide fields that reflection cannot infer:

```go
mizuoai.Post(srv, "/users", handleCreateUser,
	mizuoai.WithOperationRequestBody(requestBody),
	mizuoai.WithResponse(http.StatusCreated, createdResponse),
	mizuoai.WithResponse(http.StatusBadRequest, badRequestResponse),
)
```

`WithOperation` accepts a complete `*v3.Operation` and disables typed
reflection for that registration. `WithOaiComponents` adds any of the OpenAPI
3.2 component namespaces, including schemas, responses, parameters, examples,
request bodies, headers, security schemes, links, callbacks, path items, and
media types.

`WithOperationRequestBody` and `WithOperationResponses` replace reflected
request-body and response defaults. `WithResponse` composes additional status
codes with the reflected success response.

An authoritative operation can be supplemented with fields or named entries it
does not define. For example, `WithResponse` can add another status code. An
option that supplies a different value for an existing summary, parameter,
response code, callback, extension, or other named entry fails registration.
The source operation is never mutated.

## Raw handlers

Raw registration is intended for multipart uploads, binary downloads, SSE,
WebSocket upgrades, proxies, and other handlers where `Rx.MizuRead` or
`Tx.MizuWrite` is not the transport implementation.

```go
mizuoai.PostRaw(srv, "/upload", handleUpload,
	mizuoai.WithOperation(uploadOperation),
)
```

Raw registration passes the `http.ResponseWriter` and `*http.Request` received
after Mizu middleware directly to the handler. It does not read or replace the
body, wrap the writer, infer content, delay headers, or interfere with flushing
and upgrades.

Helpers are available for every HTTP method currently supported by Mizu:

- `GetRaw`
- `PostRaw`
- `PutRaw`
- `DeleteRaw`
- `PatchRaw`
- `HeadRaw`
- `OptionsRaw`
- `TraceRaw`
- `ConnectRaw`

## OpenAPI operation input

Parse a generated OpenAPI 3.0, 3.1, or 3.2 document once and select operations
by their unique operation ID:

```go
doc, err := mizuoai.ParseOpenAPI(registryv1.OPENAPI)
if err != nil {
	panic(err)
}

mizuoai.PostRaw(
	srv.Group("/registry"),
	"/scopes/{scope}/skills",
	handleRegisterSkill,
	mizuoai.WithOpenApiOperation(doc, "registerSkill"),
)
```

Registration checks that the imported method and fully prefixed path match the
runtime route. Imported path metadata and component namespaces are preserved.
Missing or duplicate operation IDs, unresolved documents, duplicate routes,
and incompatible component collisions fail before the runtime route is added.

Use `MustParseOpenAPI` for immutable embedded documents that should panic on an
invalid generated artifact during process initialization.

Imported artifacts retain the complete raw OpenAPI representation while
`libopenapi` models provide routing and typed construction. This preserves the
full OpenAPI 3.2 surface, including fields not yet represented by libopenapi's
high-level Go structs, such as `prefixEncoding`, nested `itemEncoding`,
`deviceAuthorization`, and `deviceAuthorizationUrl`.

## Upload, download, and SSE

Uploads use an ordinary `multipart/form-data` request body. OpenAPI 3.1 and 3.2
describe a binary part through its media type and JSON Schema content keywords:

```yaml
requestBody:
  required: true
  content:
    multipart/form-data:
      schema:
        type: object
        required: [name, package]
        properties:
          name:
            type: string
          package:
            contentMediaType: application/gzip
```

Downloads use an ordinary response media type and may document headers such as
`Content-Disposition`:

```yaml
responses:
  "200":
    description: Compressed package
    headers:
      Content-Disposition:
        required: true
        schema:
          type: string
    content:
      application/gzip: {}
```

OpenAPI 3.2 models each item in an SSE stream with `itemSchema`:

```yaml
responses:
  "200":
    description: Event stream
    content:
      text/event-stream:
        itemSchema:
          $ref: "#/components/schemas/Event"
```

If an SSE `data` field contains encoded JSON, its string schema can use
`contentMediaType: application/json` and `contentSchema`.

## OpenAPI configuration

```go
mizuoai.Initialize(srv, "Service API",
	mizuoai.WithOaiVersion("3.2.0"),
	mizuoai.WithOaiInfoVersion("v1"),
	mizuoai.WithOaiSummary("Service API"),
	mizuoai.WithOaiDescription("HTTP API for the service"),
	mizuoai.WithOaiSelf("https://example.com/openapi.yaml"),
	mizuoai.WithOaiJsonSchemaDialect("https://json-schema.org/draft/2020-12/schema"),
	mizuoai.WithOaiDocumentation(),
	mizuoai.WithOaiServePath("/docs"),
)
```

YAML is served by default. `WithOaiRenderJson` switches the endpoint to JSON.
`WithOaiPreLoad` accepts a complete OpenAPI 3.0, 3.1, or 3.2 input document;
3.1 input is the primary compatibility path. Output is normalized to the
configured target version, which defaults to 3.2.0. Output version 3.1.x is
also available through `WithOaiVersion` for older downstream tooling; 3.2-only
fields are rejected when that target is selected.

For OpenAPI 3.0 input, `nullable` and boolean exclusive-bound semantics are
translated to their OpenAPI 3.1/3.2 JSON Schema equivalents, so a plugin pinned
to OpenAPI 3.0.2 does not need to be upgraded before adopting this package.
Current `sudorandom-connect-openapi` releases generate OpenAPI 3.1.0, so their
output can also be imported directly without asking the plugin to generate
3.2.

## Validation behavior

Every input and rendered document is validated against the official OpenAPI
3.0, 3.1, or 3.2 schema embedded by `libopenapi-validator`, then parsed and
rebuilt with `libopenapi`. Registration completes this validation before adding
the runtime route. It also rejects:

- duplicate method/path pairs;
- duplicate non-empty operation IDs;
- missing or non-required path parameters;
- imported method/path mismatches;
- operations without responses;
- incompatible components with the same namespace and name.

Security requirements must name declared security schemes. Tag names must be
unique, and OpenAPI 3.2 tag parents must exist without forming a cycle.

The OpenAPI document builder merges methods into one Path Item, so GET and POST
can safely share a route path. Rendering starts from a fresh model each time and
is deterministic for an identical registration sequence.

## Migration notes

- The generated version is now OpenAPI 3.2.0 instead of 3.0.2. Use
  `WithOaiVersion("3.1.1")` only when a downstream tool still requires 3.1.
- Paths are emitted as real Path Item Objects. Different methods share one path
  key; the former `"METHOD /path"` shape is no longer produced.
- Named Go types now become `components.schemas` entries referenced with
  `$ref`. Use `WithOaiSchemaName[T]` for a stable external component name.
- JSON tags are canonical for request locations and parameter names. Matching
  legacy tags remain accepted, but conflicting legacy and JSON tags now fail.
- Reflected `[]byte` remains base64 for JSON. Raw binary bodies use their media
  type without a JSON schema, while multipart byte fields use
  `contentMediaType` without claiming base64 transport encoding.
- Invalid documents, unresolved references, duplicate routes, and component
  collisions now fail during initialization or route registration instead of
  being printed or deferred until traffic is served.
- Stoplight Elements is pinned to 9.0.24 so documentation UI behavior does not
  drift independently of the package.
